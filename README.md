# FitFindr

## Overview

FitFindr is a multi-tool AI agent that helps users discover secondhand clothing and figure out how to style it with their existing wardrobe. The agent searches a mock thrift marketplace dataset, generates outfit suggestions using the user's wardrobe, and creates a shareable social-media-style fit card.

The system uses three tools connected through a planning loop and passes information between tools through a shared session state.

---

# Tool Inventory

## 1. search_listings(description, size, max_price)

### Purpose

Searches the mock listings dataset and finds secondhand items that match the user's request.

### Inputs

* `description (str)` – Item description or keywords (e.g., "vintage graphic tee")
* `size (str | None)` – Desired clothing size
* `max_price (float | None)` – Maximum budget

### Returns

A `list[dict]` of matching listings sorted by relevance.

Each listing contains:

* id
* title
* description
* category
* style_tags
* size
* condition
* price
* colors
* brand
* platform

### Failure Handling

If no matching listings are found, the function returns an empty list instead of raising an exception.

---

## 2. suggest_outfit(new_item, wardrobe)

### Purpose

Creates outfit suggestions using the selected thrifted item and the user's wardrobe.

### Inputs

* `new_item (dict)` – Listing selected from search results
* `wardrobe (dict)` – User wardrobe containing wardrobe items

### Returns

A string containing one or more outfit recommendations.

### Failure Handling

If the wardrobe is empty, the function provides general styling advice rather than crashing or returning an empty response.

---

## 3. create_fit_card(outfit, new_item)

### Purpose

Creates a short social-media-style caption describing the outfit.

### Inputs

* `outfit (str)` – Outfit suggestion generated by suggest_outfit
* `new_item (dict)` – Selected listing

### Returns

A 2–4 sentence caption suitable for Instagram or TikTok.

### Failure Handling

If the outfit string is empty, the function returns a descriptive error message rather than raising an exception.

---

# Planning Loop

The agent follows a conditional planning loop rather than calling every tool automatically.

1. Parse the user query into:

   * description
   * size
   * max_price

2. Call `search_listings()`.

3. Check search results:

   * If results are empty:

     * Set an error message.
     * Return immediately.
     * Do not call the remaining tools.
   * Otherwise:

     * Store the top result as `selected_item`.

4. Call `suggest_outfit(selected_item, wardrobe)`.

5. Store the outfit suggestion.

6. Call `create_fit_card(outfit_suggestion, selected_item)`.

7. Return the completed session.

This allows the agent to behave differently depending on what happens during the search phase.

---

# State Management

The agent uses a shared session dictionary to store information between tool calls.

Stored state:

```python
{
    "query": query,
    "parsed": {},
    "search_results": [],
    "selected_item": None,
    "wardrobe": wardrobe,
    "outfit_suggestion": None,
    "fit_card": None,
    "error": None
}
```

State flow:

1. `search_listings()` returns results.
2. Top result is stored in `selected_item`.
3. `selected_item` is passed directly into `suggest_outfit()`.
4. `suggest_outfit()` returns an outfit suggestion.
5. Outfit suggestion is stored in `outfit_suggestion`.
6. `outfit_suggestion` is passed directly into `create_fit_card()`.
7. Final caption is stored in `fit_card`.

The user never has to re-enter information between steps.

# Error Handling

## search_listings Failure

### Trigger

```text
designer ballgown size XXS under $5
```

### Response

```text
I couldn't find any listings for that query.
Try using a broader search term, increasing your budget, or removing the size filter.
```

The planning loop stops and does not call additional tools.

---

## suggest_outfit Failure

### Trigger

Empty wardrobe.

### Response

Instead of failing, the tool provides general styling advice using common wardrobe pieces.

---

## create_fit_card Failure

### Trigger

Empty outfit string.

### Response

```text
I couldn't create a fit card because the outfit suggestion was empty.
Try generating an outfit first.
```

# Spec Reflection

### How the specification helped

The planning.md specification helped define each tool's interface before implementation. This made it easier to implement and test each tool independently before connecting them through the planning loop.

### One divergence from the specification

The specification allowed query parsing through regex, string processing, or an LLM. I chose simple regex-based parsing because it was easier to test, required fewer API calls, and provided predictable behavior for extracting price and size information.
